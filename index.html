Executive summary

What the code does and what it produced

ExoMetrix trains a two-stage ML pipeline for real-time gait analysis. The code reads raw goniometer (joint angle) data from the Camargo et al. AB subject biomechanics dataset, extracts signals from MATLAB files, calibrates them to a clinical range, then trains three models that ship together as a single .joblib artifact.

Three models trained

1Anomaly detector — IsolationForest
Flags unusual gait angles as compensating steps. Trained on 868K calibrated angle samples with RobustScaler preprocessing.
2Activity classifier — RandomForest
Identifies locomotion type: levelground, ramp, stair, treadmill. Trained on 14-feature window vectors with 434 labeled examples.
3Intention classifier — RandomForest
Predicts movement intent: walking, running, upstairs, ramp_up. Same feature matrix as activity classifier, different labels.

Runtime outputs per prediction call

Step quality classification · Assistance percentage (0–95%) · Anomaly score (0–1) · Activity class · Intention class · Cadence (steps/min) · Toe clearance (mm) · Gait phase · Model confidence

Step 1 — argument parsing

The CLI accepts --dataset-root, --subject-glob, --output-model, and --output-meta. Defaults assume the training script lives two levels below the dataset root.

Step 2 — subject discovery

Checks whether dataset_root is already a single subject folder (detected by looking for an osimxml/ child) or a parent of multiple subjects. Globs for AB* folders containing osimxml/. Works for training on AB06 alone or AB06+AB07+AB08 transparently.

Step 3 & 4 — MAT loading and signal extraction

Each gon/*.mat file is loaded with scipy first, then h5py as a fallback for MATLAB v7.3 HDF5 files. Signal extraction tries normal numpy array traversal first, then falls back to parsing the MATLAB opaque workspace blob as raw float64 bytes. Signals are clipped to −220…+220° and downsampled to at most 2000 samples per file.

Step 5 — global calibration

All subjects' angles are pooled and linearly rescaled so the 1st–99th percentile range maps to 5°–120° (the typical clinical gait range). Values are hard-clipped to 0–180°. This ensures the anomaly model trains on a clinically meaningful scale.

Step 6 — anomaly model training

Calibrated angles are reshaped to (N, 1) and standardized with RobustScaler. An IsolationForest with 300 trees and 8% contamination is trained. The 1st and 5th percentile of the resulting decision scores are stored as normalization bounds used at runtime.

Step 7 — window features

The script iterates the same MAT files a second time. Each file's full signal is treated as one window, and 14 statistical features are computed from it. Labels come from the file's directory path.

Step 8 — classifier training

Both RandomForest classifiers receive the same 434-row feature matrix. An 80/20 stratified split validates accuracy. Both scored 93.1% on the 87-window holdout.

Step 9 — artifact saving

Everything is packed into a single dict and saved with joblib.dump(). A JSON metadata file is also written for the Flask backend status endpoint.

    Feature engineering

    The 14 statistical features computed per window

    Each MAT file is treated as one signal window. 14 statistics are extracted — 8 positional, 4 velocity-based, 1 energy, 1 temporal structure metric. Velocity is computed as first differences of sample values, not time-normalized.

    mean_angle
    μ = (1/N) Σ xᵢ
    Average joint position — separates treadmill (low) from stair (high)
    std_angle
    σ = √Var(x)
    Motion spread. High for stairs, low for constrained treadmill
    min_angle
    min(x)
    Minimum joint position during window
    max_angle
    max(x)
    Stair needs high max; asymmetry encodes ramp
    range_angle
    max(x) − min(x)
    Total arc. Reduced range may indicate stiffness
    median_angle
    x̃ = pct(x, 50)
    Robust central tendency — differs from mean on skewed ramp signals
    p10_angle
    pct(x, 10)
    Robust lower bound, resistant to outliers
    p90_angle
    pct(x, 90)
    Robust upper bound, resistant to outliers
    mean_abs_velocity
    (1/N−1) Σ |Δxᵢ|
    Average angular speed. Running → high; slow walking → low
    std_velocity
    σ(Δx)
    Speed irregularity. High for stairs, low for treadmill
    max_velocity
    max(Δx)
    Peak forward angular acceleration
    min_velocity
    min(Δx)
    Peak reverse angular acceleration
    energy
    (1/N) Σ (xᵢ−μ)²
    Actually variance — redundant with std² but kept for naming clarity
    peak_density
    peaks / N, t=μ+0.25σ
    Approximate step frequency. Irregular gait → unusual density

    Note Velocity features are not time-normalized. Files are downsampled at different strides (step = size ÷ 2000), so velocity units differ across files — a known design issue.

    Labeling and dataset

    Where labels come from and where they may be wrong

    Labels come entirely from file system paths — there is no human-annotated label file. The grandparent folder name gives the activity, and the filename stem provides intent signals.

    Folder Activity class Intention class How decided
    levelground/ levelground walking or running _fast_ in filename → running; else walking
    stair/ stair upstairs Always upstairs — descent not captured
    ramp/ ramp ramp_up Always ramp_up — descent not captured
    treadmill/ treadmill walking Always walking — speed variation not captured
    anything else unknown walking Fallback; unknown class not used by classifiers

    Bias and noise concerns

    HighStair and ramp descent are absent
    All stair files → "upstairs", all ramp files → "ramp_up" regardless of actual direction. A patient descending stairs will be confidently misclassified.
    MediumRunning detection is fragile
    Only files with _fast_ in the filename get labelled as running. Renamed files silently become "walking". Only 31 windows currently — the class is thin.
    LowHealthy young adult population only
    3 able-bodied subjects from the AB cohort. No pathological gait, elderly movement, or post-surgical compensation in training data.

    Model training decisions

    Why each algorithm and parameter was chosen

    Why RobustScaler?

    Gait angle data contains real physiological outliers — sudden jerks, sensor spikes, non-physiological MATLAB fill values. RobustScaler uses the median and IQR instead of mean and std, so spike values don't distort the scale. StandardScaler would pull the scale toward outliers and make the IsolationForest treat normal gait as unusual.

    Why IsolationForest? Why 8% contamination?

    IsolationForest works by randomly splitting features and measuring how quickly a point can be isolated. Genuine outliers get isolated in very few splits; normal inliers need many. It suits this case because: training data has no anomaly labels, it's fast for embedded deployment, and it handles the asymmetric joint-angle distribution well.

    The contamination=0.08 value tells the model to expect ~8% of training samples to be outliers. Since the AB subjects are healthy, this effectively flags "8% of healthy gait as edge-cases worth flagging." A lower value (0.03–0.05) would reduce false positives on real patients.

    Why RandomForest? Why 350 trees?

    RandomForest is robust for small datasets (434 windows) and provides probability estimates needed for confidence scoring. class_weight="balanced_subsample" resamples class weights at each tree independently, handling the ramp/treadmill imbalance. 350 trees is higher than needed — diminishing returns likely set in around 100–150 with 434 samples and 14 features.

    When is validation skipped?

    Validation is skipped when total samples < 30 or any single class has fewer than 2 members (stratified split requires ≥2 per class in both sets). In the current snapshot neither applies, so validation was performed — 87-window holdout, single split, 93.1% accuracy.

    The anomaly score normalization

    Raw IsolationForest scores have no natural 0–1 scale. The code stores training score percentiles p01 and p05. At runtime: anomaly_strength = clip((p05 − score) / (p05 − p01), 0, 1). Current values: p01=−0.113, p05=−0.027 → denominator ≈ 0.086. This narrow band makes scores near the boundary sensitive to small fluctuations.

    Relationship between anomaly model and classifiers

    The two stages are architecturally independent but semantically complementary. The IsolationForest scores a single angle per call: "is this angle consistent with normal gait?" The RandomForests score a 14-feature window: "what kind of activity is this?" A compensating step can occur during any activity — the classifiers still assign the correct activity even when the anomaly detector fires. Both run in parallel inside predict().

    Audience summaries

    The same system explained two different ways

    For clinicians

    ExoMetrix learns what normal walking, stair climbing, and ramp walking look like from joint angle data collected on healthy young adults. When a patient uses the system, it compares their joint angles against this learned normal and flags unusual patterns as "compensating steps." It also estimates the type of movement the patient is doing and how much assistance the exoskeleton should provide.

    Currently the system has been trained on only three healthy volunteers. It has not been tested on patients recovering from injury or surgery. Estimates for toe clearance and walking phase are approximations, not clinical measurements. Before clinical deployment, the system should be validated against real patient data and outputs — particularly step quality — should be reviewed by a clinician familiar with the patient population.

    For engineers

    Two-stage architecture: unsupervised anomaly detector (IsolationForest on univariate calibrated angle, RobustScaler) trained on 868K samples, paired with two supervised classifiers (RandomForest, 350 trees, balanced_subsample) trained on 14 hand-crafted stats per-file window (434 total). Labels are path-derived — no annotation file.

    Main technical risks: (1) one vector per file vs. the 120-sample runtime window — training and inference features are computed at different temporal scales; (2) velocity features not time-normalized due to variable downsampling strides; (3) anomaly score normalization band of 0.086 is narrow and noisy near the boundary; (4) 93.1% accuracy is a single-split estimate on 87 samples. Priority fixes: sliding-window extraction, k-fold CV, time-normalized velocity, and descent locomotion labels.
